# Mako Coders — 52-Week Module Plan

> *One-year, once-a-week coding curriculum for Kenyan learners aligned to CBE ICT / Computer Studies strands. Each session is 2 hours with a 10-minute break. Final project: Mako Shark Expedition.*

---

## Progression Overview

| Term | Weeks | Focus | Tool | Capstone milestone |
|------|-------|-------|------|-------------------|
| **Term 1 — Ocean Foundations** | 1–13 | Sequencing, variables, loops, conditionals, events, debugging | Scratch 3 / micro:bit MakeCode | Animated ocean scene |
| **Term 2 — Data & Coastal Systems** | 14–26 | Lists, data types, collecting data, simple visualisation, file I/O | Scratch + Python + Turtle | Marine species data explorer |
| **Term 3 — Hardware & Networks** | 27–39 | Micro:bit sensors, physical computing, networking basics, APIs | micro:bit + Python | Tide-guard sensor station (mock) |
| **Term 4 — Conservation & Capstone** | 40–52 | Functions, classes, ethics, cybersecurity, project cycle | Python + Pygame Zero | **Mako Shark Expedition** |

## Lesson Time Structure (2 hours)

| Segment | Duration | Purpose |
|---------|----------|---------|
| Arrival & setup | 10 min | Log in, open project, settle in |
| Hook — marine story | 10 min | Introduce species, conservation hook, inquiry question |
| Concept mini-lesson | 20 min | Teach the coding idea with a live demo |

## Capstone Requirements (Mako Shark Expedition)

A learner must produce one of these formats:

1. **Pygame Zero game** — a top-down or side-view ocean where the player controls a mako shark, collects data, and avoids threats.
2. **Text-mode dashboard** — a Python CLI that loads species data and lets the user explore facts, sightings, and conservation tips.
3. **Scratch advanced project** — for learners without Python access; must include custom blocks, lists, and a conservation message.

Every capstone must include:

- At least one real Kenyan marine species with correct name and conservation note.
- User input (keyboard, mouse, or typed command).
- A loop.
- A conditional.
- A function or custom block.
- A list or dictionary.
- A conservation message.
- A peer presentation.
