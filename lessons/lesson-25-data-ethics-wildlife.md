# Lesson 25: Data Ethics — Keep Wildlife Safe

- **Module Reference:** Prompt 25
- **Duration:** 2 hours (10-minute break after guided practice)
- **Age Group:** CBE Junior/Senior School (12-15 years)
- **Coding Tool:** Discussion + Python

## CBE Strand Mapping

| CBE Learning Area | Strand | Sub-strand |
|-------------------|--------|------------|
| Computer Studies / ICT | Citizenship | Digital Citizenship and Conservation |

## Learning Outcomes

By the end of the lesson the learner should be able to:
1. **Explain** explain how technology affects communities and nature.
2. **Apply** apply ethical choices about data and conservation.
3. **Value** value using code for the common good.

## Key Inquiry Question

*How can the All species (viumbe vya baharini) help us learn about privacy, location safety, and responsible data use?*

## Resources

- Discussion + Python installed on learner devices (offline setup where possible)
- Projector for live demo
- Starter files or handouts as needed
- Reference material from `research/domain-reference.md`

## Local Context: All species

- **Common name:** All species
- **Swahili name:** viumbe vya baharini
- **Habitat:** Kenyan coastal and marine ecosystems
- **Why this species:** This species anchors the week's concept because its behaviour or status connects naturally to privacy, location safety, and responsible data use.
- **Conservation note:** all marine life benefits from protected areas, clean coasts, and responsible fishing.

## Lesson Development

### Intro hook (10 min)
Share a quick fact about the All species (viumbe vya baharini). Ask a question that links the species to the coding concept: "How can we use code to safe-data...?"

### Step 1 — Concept + teacher demo (20 min)
Teach the week's concept (privacy, location safety, and responsible data use) using a short live demo in Discussion + Python. Connect every block or line of code back to the All species.

### Step 2 — Guided practice (30 min)
Learners follow along with a starter activity. They make small changes and test frequently. The teacher circulates and uses the anchor species as a memory hook.

### Break (10 min)

### Step 3 — Independent practice (35 min)
Create the main deliverable: **safe-data pledge**. Checklist:
- [ ] Core concept from the lesson is present
- [ ] Anchor species (All species) is referenced in the project
- [ ] Code runs without critical errors
- [ ] One conservation message or note is included

### Step 4 — Sharing and feedback (15 min)
Two or three learners demo their work. Peers give one "glow" (what worked well) and one "grow" (one idea to improve) using CBE rubric language.

Research one additional fact about the All species and add it to the project. Alternatively, modify the code to respond to a second input or display extra data.
