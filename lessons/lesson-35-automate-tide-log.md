# Lesson 35: Automate the Tide Log

- **Module Reference:** Prompt 35
- **Duration:** 2 hours (10-minute break after guided practice)
- **Age Group:** CBE Junior/Senior School (12-15 years)
- **Coding Tool:** Python

## Learning Outcomes

By the end of the lesson the learner should be able to:
1. **Explain** explain the sequence of instructions.
2. **Apply** apply control structures to solve a problem.
3. **Value** value clear, ordered thinking in code.

## Key Inquiry Question

*How can the Seagrass beds (mwani wa baharini) help us learn about automation with loops and file logging?*

## Resources

- Python installed on learner devices (offline setup where possible)
- Projector for live demo
- Starter files or handouts as needed

## Local Context: Seagrass beds

- **Common name:** Seagrass beds
- **Swahili name:** mwani wa baharini
- **Habitat:** shallow coastal seabed; nursery for fish and food for turtles/dugongs
- **Why this species:** This species anchors the week's concept because its behaviour or status connects naturally to automation with loops and file logging.
- **Conservation note:** threatened by salt pans, shrimp farms, sand mining, and siltation.

## Lesson Development

### Intro hook (10 min)
Share a quick fact about the Seagrass beds (mwani wa baharini). Ask a question that links the species to the coding concept: "How can we use code to daily...?"

### Step 1 — Concept + teacher demo (20 min)
Teach the week's concept (automation with loops and file logging) using a short live demo in Python. Connect every block or line of code back to the Seagrass beds.

### Step 2 — Guided practice (30 min)
Learners follow along with a starter activity. They make small changes and test frequently. The teacher circulates and uses the anchor species as a memory hook.

### Break (10 min)

### Step 3 — Independent practice (35 min)
Create the main deliverable: **daily tide log**. Checklist:
- [ ] Core concept from the lesson is present
- [ ] Anchor species (Seagrass beds) is referenced in the project
- [ ] Code runs without critical errors
- [ ] One conservation message or note is included

### Step 4 — Sharing and feedback (15 min)
Two or three learners demo their work. Peers give one "glow" (what worked well) and one "grow" (one idea to improve) using CBE rubric language.

## Extended Activity

Research one additional fact about the Seagrass beds and add it to the project. Alternatively, modify the code to respond to a second input or display extra data.

## Extra Credit Challenge

**Schedule a log.** Make your program append a new line to a file every time it runs, with a timestamp.

## Homework Assignment

Write down one chore you do daily. List the steps and identify one that a computer could remind you about.
