# Lesson 10: Debug the Mangrove

- **Module Reference:** Prompt 10
- **Duration:** 2 hours (10-minute break after guided practice)
- **Age Group:** CBE Junior/Senior School (12-15 years)
- **Coding Tool:** Scratch 3

## CBE Strand Mapping

| CBE Learning Area | Strand | Sub-strand |
|-------------------|--------|------------|
| Computer Studies / ICT | Critical Thinking | Problem Solving and Debugging |

## Learning Outcomes

By the end of the lesson the learner should be able to:
1. **Explain** explain how to find and fix errors.
2. **Apply** apply debugging strategies to broken code.
3. **Value** value persistence when solving problems.

## Key Inquiry Question

*How can the Mangroves (mikoko) help us learn about reading error messages and fixing bugs step by step?*

## Resources

- Scratch 3 installed on learner devices (offline setup where possible)
- Projector for live demo
- Starter files or handouts as needed
- Reference material from `research/domain-reference.md`

## Local Context: Mangroves

- **Common name:** Mangroves
- **Swahili name:** mikoko
- **Habitat:** intertidal coastal forests and estuaries
- **Why this species:** This species anchors the week's concept because its behaviour or status connects naturally to reading error messages and fixing bugs step by step.
- **Conservation note:** overharvested for charcoal and firewood; restoration protects coasts and fisheries.

## Lesson Development

### Intro hook (10 min)
Share a quick fact about the Mangroves (mikoko). Ask a question that links the species to the coding concept: "How can we use code to working...?"

### Step 1 — Concept + teacher demo (20 min)
Teach the week's concept (reading error messages and fixing bugs step by step) using a short live demo in Scratch 3. Connect every block or line of code back to the Mangroves.

### Step 2 — Guided practice (30 min)
Learners follow along with a starter activity. They make small changes and test frequently. The teacher circulates and uses the anchor species as a memory hook.

### Break (10 min)

### Step 3 — Independent practice (35 min)
Create the main deliverable: **working mangrove animation**. Checklist:
- [ ] Core concept from the lesson is present
- [ ] Anchor species (Mangroves) is referenced in the project
- [ ] Code runs without critical errors
- [ ] One conservation message or note is included

### Step 4 — Sharing and feedback (15 min)
Two or three learners demo their work. Peers give one "glow" (what worked well) and one "grow" (one idea to improve) using CBE rubric language.

| Completes the main task | Cannot run starter code | Makes 1–2 guided changes | Completes the deliverable with testing | Extends the deliverable with an original feature |
| Conservation connection | No species link | Mentions species but no message | Includes a relevant conservation note | Connects conservation note to the code purpose |

## Extended Activity

Research one additional fact about the Mangroves and add it to the project. Alternatively, modify the code to respond to a second input or display extra data.

## Extra Credit Challenge

**Bug bounty.** Create a short program with one deliberate bug, swap with a partner, and see who can find and fix it fastest.

## Homework Assignment

Write down one mistake you made while coding today and how you fixed it. Bring it to share.
