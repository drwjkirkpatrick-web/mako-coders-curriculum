# Lesson 01: Meet the Mako Coder

- **Module Reference:** Prompt 01
- **Duration:** 2 hours (10-minute break after guided practice)
- **Age Group:** CBE Junior/Senior School (12-15 years)
- **Coding Tool:** Scratch 3

## CBE Strand Mapping

| CBE Learning Area | Strand | Sub-strand |
|-------------------|--------|------------|
| Computer Studies / ICT | Digital Literacy | Digital Literacy and Safety |

## Learning Outcomes

By the end of the lesson the learner should be able to:
1. **Explain** explain safe and effective computer use.
2. **Apply** apply file, editor, and internet skills.
3. **Value** value digital tools for learning.

## Key Inquiry Question

*How can the Shortfin mako shark (papa mako) help us learn about digital footprint, safe setup, file organisation, and using computers responsibly?*

## Resources

- Scratch 3 installed on learner devices (offline setup where possible)
- Projector for live demo
- Starter files or handouts as needed
- Reference material from `research/domain-reference.md`

## Local Context: Shortfin mako shark

- **Common name:** Shortfin mako shark
- **Swahili name:** papa mako
- **Habitat:** tropical and temperate Indian Ocean waters; juveniles near coast, adults offshore
- **Why this species:** This species anchors the week's concept because its behaviour or status connects naturally to digital footprint, safe setup, file organisation, and using computers responsibly.
- **Conservation note:** vulnerable globally; often taken as bycatch and targeted catch.

## Lesson Development

### Intro hook (10 min)
Share a quick fact about the Shortfin mako shark (papa mako). Ask a question that links the species to the coding concept: "How can we use code to folders,...?"

### Step 1 — Concept + teacher demo (20 min)
Teach the week's concept (digital footprint, safe setup, file organisation, and using computers responsibly) using a short live demo in Scratch 3. Connect every block or line of code back to the Shortfin mako shark.

### Step 2 — Guided practice (30 min)
Learners follow along with a starter activity. They make small changes and test frequently. The teacher circulates and uses the anchor species as a memory hook.

### Break (10 min)

### Step 3 — Independent practice (35 min)
Create the main deliverable: **folders, rules poster**. Checklist:
- [ ] Core concept from the lesson is present
- [ ] Anchor species (Shortfin mako shark) is referenced in the project
- [ ] Code runs without critical errors
- [ ] One conservation message or note is included

### Step 4 — Sharing and feedback (15 min)
Two or three learners demo their work. Peers give one "glow" (what worked well) and one "grow" (one idea to improve) using CBE rubric language.

## Assessment

| Criterion | BE (0–39%) | AE (40–59%) | ME (60–79%) | EE (80–100%) |
|-----------|------------|-------------|---------------|----------------|
| Understands the concept | Needs help to identify the concept | Identifies the concept with prompting | Explains the concept in own words | Explains and teaches the concept to a peer |
| Completes the main task | Cannot run starter code | Makes 1–2 guided changes | Completes the deliverable with testing | Extends the deliverable with an original feature |
| Conservation connection | No species link | Mentions species but no message | Includes a relevant conservation note | Connects conservation note to the code purpose |

## Extended Activity

Research one additional fact about the Shortfin mako shark and add it to the project. Alternatively, modify the code to respond to a second input or display extra data.

| Support | Advanced |
|---------|----------|
