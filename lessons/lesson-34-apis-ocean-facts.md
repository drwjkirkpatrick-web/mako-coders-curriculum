# Lesson 34: APIs — Fetching Ocean Facts

- **Module Reference:** Prompt 34
- **Duration:** 2 hours (10-minute break after guided practice)
- **Age Group:** CBE Junior/Senior School (12-15 years)
- **Coding Tool:** Python

## CBE Strand Mapping

| CBE Learning Area | Strand | Sub-strand |
|-------------------|--------|------------|
| Computer Studies / ICT | Networks | Networks and Communications |

## Learning Outcomes

By the end of the lesson the learner should be able to:
1. **Explain** explain how devices share data.
2. **Apply** apply simple networking code to send/receive messages.
3. **Value** value connected devices for marine research.

## Key Inquiry Question

*How can the Whale shark (papa mafia) help us learn about APIs and JSON data (offline-safe mock)?*

## Resources

- Python installed on learner devices (offline setup where possible)
- Projector for live demo
- Starter files or handouts as needed
- Reference material from `research/domain-reference.md`

## Local Context: Whale shark

- **Common name:** Whale shark
- **Swahili name:** papa mafia
- **Habitat:** coastal and offshore; seasonal visitor to Kenyan waters
- **Why this species:** This species anchors the week's concept because its behaviour or status connects naturally to APIs and JSON data (offline-safe mock).
- **Conservation note:** endangered; boat strikes and irresponsible tourism are major threats.

## Lesson Development

### Intro hook (10 min)
Share a quick fact about the Whale shark (papa mafia). Ask a question that links the species to the coding concept: "How can we use code to offline...?"

### Step 1 — Concept + teacher demo (20 min)
Teach the week's concept (APIs and JSON data (offline-safe mock)) using a short live demo in Python. Connect every block or line of code back to the Whale shark.

### Step 2 — Guided practice (30 min)
Learners follow along with a starter activity. They make small changes and test frequently. The teacher circulates and uses the anchor species as a memory hook.

### Break (10 min)

### Step 3 — Independent practice (35 min)
Create the main deliverable: **offline ocean-fact fetcher**. Checklist:
- [ ] Core concept from the lesson is present
- [ ] Anchor species (Whale shark) is referenced in the project
- [ ] Code runs without critical errors
- [ ] One conservation message or note is included

### Step 4 — Sharing and feedback (15 min)
Two or three learners demo their work. Peers give one "glow" (what worked well) and one "grow" (one idea to improve) using CBE rubric language.

## Extended Activity

Research one additional fact about the Whale shark and add it to the project. Alternatively, modify the code to respond to a second input or display extra data.
